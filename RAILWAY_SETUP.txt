# Instrucciones de Despliegue en Railway

## ✅ Tu app está lista para Railway

He configurado la aplicación para desplegar **todo junto en un único servicio** de Railway:

### Cambios realizados:

1. **Backend actualizado** (`backend/index.js`):
   - Ahora sirve los archivos estáticos compilados del frontend desde `/frontend/dist`
   - Redirige todas las rutas no-API a `index.html` (soporte para SPA)

2. **Dockerfile mejorado**:
   - Copia todo el código fuente (frontend y backend)
   - Compila el frontend primero
   - Instala dependencias del backend
   - Inicia el servidor en puerto 3000

3. **Configuración Railway**:
   - `Procfile`: Define cómo ejecutar la app
   - `Dockerfile`: Imagen Docker completa para despliegue

4. **Variables de entorno**:
   - `.env.example`: Template con todas las variables necesarias
   - NO commitees archivos `.env*` locales (ya están en .gitignore)
   - Railway lee las variables desde su dashboard

### 📋 Pasos para desplegar en Railway:

1. **Push tu código a GitHub** (asegúrate de que Dockerfile y backend/package.json estén en el root)

2. **Crea un proyecto en Railroad** (railway.app):
   - Conecta tu repositorio Git
   - Railway detectará automáticamente el Dockerfile

3. **Añade PostgreSQL** como servicio:
   - Railway automáticamente asignará `DATABASE_URL`

4. **Configura variables de entorno** en Railway dashboard:
   ```
   JWT_SECRET=tu-clave-segura-aqui
   NODE_ENV=production
   ```
   
   (DATABASE_URL se configura automáticamente si añades PostgreSQL)

5. **Railway compilará y deployeará automáticamente**

### 🌐 URLs después de desplegar:

- **Frontend**: `https://tu-railway-app.railway.app`
- **API**: `https://tu-railway-app.railway.app/api/...`

### ✅ Verificación después de despliegue:

- El frontend en producción usa `/api` como base (relativa)
- Las llamadas a API se harán automáticamente al mismo servidor
- No necesitas variables VITE_API_BASE_URL en producción

### Notas importantes:

- El PORT automáticamente se usa 3000 (configurable en Railway si es necesario)
- Las imágenes cargadas se almacenan en `/backend/uploads`
- Para persistencia, configura un volumen en Railway para `backend/uploads`
- El `.env` local no se commitea (seguridad) - Railway usa su dashboard para variables
